# Phase 10 Plan 01 — Core Review Infrastructure

**Phase:** 10 — Review & Revise
**Plan:** 10-01
**Date:** 2026-03-10

## Objective

Add the foundational components needed by the review() command:
- approve_review gate
- diff text generator
- corrective_actions placeholder in builder prompt

## Tasks

### Task 1: Add approve_review to approval.py

Add `approve_review(state, state_path, review_summary)` function following the same pattern as `approve_plan` but setting `review_approved`.

### Task 2: Create minilegion/core/diff.py

`generate_diff_text(execution_log: ExecutionLogSchema, project_dir: Path) -> str`

Reads each `changed_file` from the execution log and builds a human-readable diff summary:
- For create/modify: show file path, action, and content (truncated at 200 lines if needed)
- For delete: show file path and DELETE action
- Groups by task_id

Returns formatted string suitable for the reviewer prompt `{{diff_text}}` placeholder.

### Task 3: Add {{corrective_actions}} to builder.md

Add an optional section at the end of the USER_TEMPLATE:

```
{{corrective_actions}}
```

When empty string is passed, nothing extra appears. When populated (revise mode), the corrective_actions from the previous REVIEW.json are injected.

### Task 4: Tests

`tests/test_approve_review.py` (3 tests):
- approve_review sets review_approved=True on acceptance
- approve_review raises ApprovalError on rejection
- approve_review leaves STATE.json unchanged on rejection

`tests/test_diff.py` (5 tests):
- generate_diff_text returns string with file paths
- create action shows content
- delete action shows DELETE label
- empty execution log returns placeholder text
- content is truncated at 200 lines
