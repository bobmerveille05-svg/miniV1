## Tasks

### Task 1: Implement review() command

Replace `_pipeline_stub(Stage.REVIEW)` stub with full implementation:

1. `find_project_dir()` → `load_state()` → `StateMachine` → `can_transition(Stage.REVIEW)` guard
2. `load_config(project_dir.parent)` → `check_preflight(Stage.REVIEW, project_dir)`
3. Read `PLAN.json`, `DESIGN.json`, `RESEARCH.json` for reviewer context
4. Generate diff text from `EXECUTION_LOG.json` via `generate_diff_text()`
5. Extract `conventions` from RESEARCH.json (`existing_conventions` field as joined string)
6. `load_prompt("reviewer")` → `render_prompt(project_name, diff_text, plan_json, design_json, conventions)`
7. `OpenAIAdapter(config)` → `validate_with_retry(..., "review", ...)`
8. `save_dual(review_data, REVIEW.json, REVIEW.md)`
9. `approve_review(state, STATE.json, review_md)` gate
10. State transition to Stage.REVIEW + save_state()

### Task 2: Revise loop

After approval, check verdict:
- `verdict == "pass"`: done
- `verdict == "revise"`:
  - Check `state.metadata.get("revise_count", "0")` — if >= "2": escalate (print message + exit 0)
  - Check `design_conformity.conforms == False`: offer re-design prompt → if user confirms, backtrack to design stage, exit 0
  - Increment revise_count in state.metadata, save_state()
  - Re-run builder with corrective_actions injected into prompt
  - Apply patches (same per-patch approval loop as execute())
  - Save updated EXECUTION_LOG.json
  - Loop back to reviewer

### Task 3: Extract _run_builder() helper

Extract the builder LLM call + scope validate + per-patch loop + save_dual into a private `_run_builder(project_dir, config, state, corrective_actions="")` helper. Both `execute()` and the revise loop use it.
