<objective>
Sync Pydantic ResearchSchema with the JSON schema file so brainstorm fields are preserved through model_dump_json(), and add Python-level validation to enforce non-empty recommendation in brainstorm mode.

**Purpose:** Phase 16 Plan 01 updated the JSON schema and prompt for brainstorm mode, but omitted the Pydantic model update. This causes silent data loss: brainstorm fields pass Pydantic's extra-fields-allowed check, but `model_dump_json()` drops them because they're not declared. RSM-02 and RSM-03 are blocked until the Pydantic model matches the JSON schema.

**Root cause:** `ResearchSchema` in `minilegion/core/schemas.py` has only 11 fact-mode fields. The 7 brainstorm-specific fields (problem_framing, facts, assumptions, candidate_directions, tradeoffs, risks, recommendation) are present in `minilegion/schemas/research.schema.json` but not in the Pydantic model.

**Output:**
- `ResearchSchema` with 7 optional brainstorm fields added
- Post-validation check in `research()` command enforcing non-empty recommendation when `require_recommendation=True`
- Tests that bypass the `validate_with_retry` mock to exercise the real Pydantic model serialization path
</objective>
