<objective>
Implement the pre-parse fixup pipeline and retry-with-feedback logic for LLM output validation.

Purpose: LLM output is messy — responses may include markdown code fences, trailing commas, or BOM characters. The fixup pipeline normalizes this before validation. When validation still fails, retry logic re-calls the LLM with human-readable error feedback. After max retries, raw output is saved for debugging. This completes the full validation-retry loop that the LLM adapter (Phase 3) will call.

Output: fixups.py with 3 fixup functions + pipeline, retry.py with validate_with_retry + error summarization + debug file saving, comprehensive tests.
</objective>

  <files>minilegion/core/fixups.py, tests/test_fixups.py</files>
  <behavior>
    - strip_markdown_fences removes ```json...``` wrappers and returns inner JSON
    - strip_markdown_fences removes plain ``` wrappers (no language tag)
    - strip_markdown_fences returns text unchanged if no fences present
    - strip_markdown_fences handles fences with leading/trailing whitespace
    - fix_trailing_commas removes comma before } (e.g., {"a": 1,} → {"a": 1})
    - fix_trailing_commas removes comma before ] (e.g., [1, 2,] → [1, 2])
    - fix_trailing_commas handles nested trailing commas
    - fix_trailing_commas does NOT remove commas inside string values
    - strip_bom_and_control removes UTF-8 BOM (\ufeff) from start
    - strip_bom_and_control removes control characters (\x00-\x08, \x0b, \x0c, \x0e-\x1f, \x7f) but preserves \n, \r, \t
    - apply_fixups chains all 3 in correct order: BOM → fences → trailing commas
    - apply_fixups with combined input (BOM + fences + trailing commas) produces valid JSON
  </behavior>
  <action>
    Create `minilegion/core/fixups.py` with 4 functions following the exact patterns from RESEARCH.md:

    1. `strip_bom_and_control(text: str) -> str` — remove BOM prefix and control chars (preserve \n, \r, \t)
    2. `strip_markdown_fences(text: str) -> str` — regex to strip ```json...``` or plain ``` wrappers
    3. `fix_trailing_commas(text: str) -> str` — regex `r',\s*([}\]])'` → `r'\1'` to remove trailing commas before } or ]
    4. `apply_fixups(raw_text: str) -> str` — chain in order: (1) BOM/control, (2) fences, (3) trailing commas

    NOTE on fix_trailing_commas: the simple regex `r',\s*([}\]])'` will also match commas inside string values in edge cases. This is acceptable — per CONTEXT.md, keep fixups to the 3 agreed-upon patterns. Don't over-engineer. Structural JSON errors should fail validation and trigger retry.
