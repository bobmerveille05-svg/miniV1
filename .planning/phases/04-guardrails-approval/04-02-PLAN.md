<objective>
Build the 5 human approval gates that block state transitions on rejection — the interactive safety layer between artifact production and state mutation.

Purpose: Ensure no state mutation occurs without explicit human consent. Each approval gate displays a summary of the artifact, prompts Y/N via `typer.confirm()`, and either persists the approval atomically or raises `ApprovalError` without touching STATE.json. This is the critical APRV-06 guarantee: rejection = byte-identical state file.

Output: `approval.py` with `approve()` core function and 5 gate-specific wrapper functions, plus comprehensive unit tests covering approval, rejection, and byte-identical rejection behavior.
</objective>

Create `minilegion/core/approval.py`:

1. Import `typer`, `Path` from pathlib, `ProjectState` and `save_state` from `minilegion.core.state`, `ApprovalError` from `minilegion.core.exceptions`.

2. Implement `approve(gate_name: str, summary: str, state: ProjectState, state_path: Path) -> bool`:
   - `typer.echo(summary)` — display the summary to the user.
   - `approved = typer.confirm(f"Approve {gate_name}?")` — DO NOT use `abort=True`.
   - If `not approved`: `raise ApprovalError(f"Rejected: {gate_name}")` — NO state modification before this point.
   - If approved: `state.approvals[gate_name] = True`, `state.add_history("approval", f"Approved: {gate_name}")`, `save_state(state, state_path)`, `return True`.

3. Implement 5 gate-specific functions. Each formats a concise summary from the artifact content and delegates to `approve()`:

   `approve_brief(state: ProjectState, state_path: Path, brief_content: str) -> bool`:
   - Summary: `"=== Brief for Approval ===\n\n{brief_content}\n"`
   - Calls `approve("brief_approved", summary, state, state_path)`

   `approve_research(state: ProjectState, state_path: Path, research_summary: str) -> bool`:
   - Summary: `"=== Research Summary for Approval ===\n\n{research_summary}\n"`
   - Calls `approve("research_approved", summary, state, state_path)`

   `approve_design(state: ProjectState, state_path: Path, design_summary: str) -> bool`:
   - Summary: `"=== Design for Approval ===\n\n{design_summary}\n"`
   - Calls `approve("design_approved", summary, state, state_path)`

   `approve_plan(state: ProjectState, state_path: Path, plan_summary: str) -> bool`:
   - Summary: `"=== Plan for Approval ===\n\n{plan_summary}\n"`
   - Calls `approve("plan_approved", summary, state, state_path)`

   `approve_patch(state: ProjectState, state_path: Path, diff_text: str) -> bool`:
   - Summary: `"=== Patch for Approval ===\n\n{diff_text}\n"`
   - Calls `approve("execute_approved", summary, state, state_path)`

Create `tests/test_approval.py`:

**CRITICAL anti-pattern to avoid:** Do NOT use `typer.confirm(abort=True)`. It raises `typer.Abort` instead of returning False, bypassing our `ApprovalError` exception hierarchy. Always call `typer.confirm()` without `abort=True` and check the return value.

**CRITICAL for APRV-06:** The `approve()` function MUST NOT modify the `state` object or call `save_state()` before `typer.confirm()` returns True. The rejection path raises immediately — no mutation, no rollback needed.
