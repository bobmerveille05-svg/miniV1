  <action>
    **1. Create `minilegion/core/renderer.py`:**

    ```python
    """Dual-output renderer: Pydantic models → human-readable Markdown.

    Each LLM-produced artifact schema has a dedicated render function that
    produces structured Markdown. The save_dual() convenience function
    writes both JSON and Markdown atomically.
    """
    from pathlib import Path
    from pydantic import BaseModel
    from minilegion.core.file_io import write_atomic
    from minilegion.core.schemas import (
        ResearchSchema, DesignSchema, PlanSchema,
        ExecutionLogSchema, ReviewSchema,
    )
    ```

    **Helper functions** (private, for DRY rendering):
    - `_bullets(items: list[str], heading: str) -> str` — render list as `## {heading}\n\n- item\n- item\n\n`. Return empty string if items is empty.
    - `_kv(label: str, value: str) -> str` — render as `**{label}:** {value}\n\n`.

    **`render_research_md(data: ResearchSchema) -> str`:**
    - Title: `# Research Report`
    - `## Project Overview` — `data.project_overview`
    - `## Tech Stack` — bullet list from `data.tech_stack`
    - `## Architecture Patterns` — bullet list
    - `## Relevant Files` — bullet list
    - `## Existing Conventions` — bullet list
    - `## Dependencies Map` — for each key in `data.dependencies_map`, show `### {key}` with bullet list of values
    - `## Potential Impacts` — bullet list
    - `## Constraints` — bullet list
    - `## Assumptions Verified` — bullet list
    - `## Open Questions` — bullet list
    - `## Recommended Focus Files` — bullet list
    - Skip sections with empty lists/dicts

    **`render_design_md(data: DesignSchema) -> str`:**
    - Title: `# Design Document`
    - `## Design Approach` — `data.design_approach`
    - `## Architecture Decisions` — for each decision: `### {decision.decision}` with Rationale paragraph and `Rejected alternatives:` bullet list
    - `## Components` — for each component: `### {component.name}` with description and files list
    - Remaining list fields as bullet sections: Data Models, API Contracts, Integration Points, Design Patterns Used, Conventions to Follow, Technical Risks, Out of Scope
    - `## Test Strategy` — `data.test_strategy`
    - `## Estimated Complexity` — `data.estimated_complexity`

    **`render_plan_md(data: PlanSchema) -> str`:**
    - Title: `# Implementation Plan`
    - `## Objective` — `data.objective`
    - `## Design Reference` — `data.design_ref`
    - `## Assumptions` — bullet list
    - `## Tasks` — for each task: `### {task.id}: {task.name}` with description, files list, depends_on, component
    - `## Touched Files` — bullet list
    - `## Risks` — bullet list
    - `## Success Criteria` — bullet list
    - `## Test Plan` — `data.test_plan`

    **`render_execution_log_md(data: ExecutionLogSchema) -> str`:**
    - Title: `# Execution Log`
    - For each task result: `## Task: {task.task_id}` with:
      - Changed files list (path + action)
      - Unchanged files list
      - Tests run list
      - Test result
      - Blockers list
      - Out of scope needed list

    **`render_review_md(data: ReviewSchema) -> str`:**
    - Title: `# Review Report`
    - `## Verdict` — bold PASS or REVISE
    - `## Bugs` — bullet list
    - `## Scope Deviations` — bullet list
    - `## Design Conformity` — "Conforms: Yes/No" + deviations if any
    - `## Convention Violations` — bullet list
    - `## Security Risks` — bullet list
    - `## Performance Risks` — bullet list
    - `## Tech Debt` — bullet list
    - `## Out of Scope Files` — bullet list
    - `## Success Criteria Met` — bullet list
    - `## Corrective Actions` — bullet list

    **`_RENDERERS` dict:**
    ```python
    _RENDERERS: dict[str, callable] = {
        "ResearchSchema": render_research_md,
        "DesignSchema": render_design_md,
        "PlanSchema": render_plan_md,
        "ExecutionLogSchema": render_execution_log_md,
        "ReviewSchema": render_review_md,
    }
    ```

    **`save_dual(data: BaseModel, json_path: Path, md_path: Path) -> None`:**
    - `json_str = data.model_dump_json(indent=2)`
    - `write_atomic(json_path, json_str)`
    - Look up `type(data).__name__` in `_RENDERERS`; raise `ValueError(f"No renderer registered for {type_name}")` if missing
    - `md_str = renderer(data)`
    - `write_atomic(md_path, md_str)`
    - Both `json_path` and `md_path` are `Path` objects
