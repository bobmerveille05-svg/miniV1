    **1. Create `minilegion/prompts/loader.py`:**

    ```python
    import re
    from importlib import resources
    from minilegion.core.exceptions import ConfigError

    SYSTEM_MARKER = "<!-- SYSTEM -->"
    USER_TEMPLATE_MARKER = "<!-- USER_TEMPLATE -->"
    ```

    - `load_prompt(role: str) -> tuple[str, str]`:
      - Build filename as `f"{role}.md"`
      - Load via `resources.files("minilegion.prompts").joinpath(filename).read_text("utf-8")`
      - Catch `FileNotFoundError` (and also `TypeError` which `importlib.resources` may raise for missing files) → raise `ConfigError(f"Prompt file not found: {filename}")`
      - Validate both markers exist in content, raise `ConfigError` if either missing
      - Split on `USER_TEMPLATE_MARKER` first (2 parts), then split first part on `SYSTEM_MARKER`
      - `.strip()` both sections (handles Windows `\r\n`)
      - Return `(system_prompt, user_template)`

    - `render_prompt(template: str, **variables: str) -> str`:
      - Use `re.sub(r"\{\{(\w+)\}\}", replacer, template)` where `replacer` checks if key is in variables dict
      - If key not found, raise `ConfigError(f"Unresolved placeholder: {{{{{key}}}}} — available variables: {sorted(variables.keys())}")`
      - Return substituted string

    **2. Create 5 prompt files** in `minilegion/prompts/`:

    Each file follows this structure:
    ```

    You MUST respond with valid JSON only. No markdown, no explanations, no code fences.

    [Role definition and behavioral constraint]
    [Output schema field descriptions — reference the exact field names from the Pydantic model in schemas.py]
    [Any role-specific instructions]

    CRITICAL: Your entire response must be a single valid JSON object. Nothing else.

    <!-- USER_TEMPLATE -->
    [Template with {{placeholder}} variables per the CONTEXT.md spec]
    ```
