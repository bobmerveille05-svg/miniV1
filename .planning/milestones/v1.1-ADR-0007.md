# ADR-0007: Evolve miniV1 Toward a Portable Kernel

**Date:** 2026-03-11
**Status:** Accepted
**Milestone:** v1.1 Portable Kernel

---

## Context

After v1.0 shipped, three structural gaps became clear from real usage:

1. **STATE.json does too much.** It holds both current state and the full event history. This makes it hard to query "what happened" vs "where are we now", and the history section grows unbounded.

2. **No `context <tool>` command.** After any break (hours, days), a user must re-read the repo, recall which model was used, and manually assemble context before continuing. There is no one-command resume path.

3. **Research lacks exploration modes.** The current `--mode fact` scanner is good for "what does this codebase do?" but produces no candidate directions, no tradeoffs, no recommendation. When the problem space is open-ended, the researcher output gives nothing to feed into design decisions.

4. **No validate/advance separation.** Stage-producing commands auto-advance state. This means a user cannot edit an artifact and re-validate without re-running the full LLM pipeline. There is no evidence trail for what was validated and when.

5. **No rollback.** If a design is rejected after review, the only option is manual STATE.json editing. There is no safe, auditable way to reset to a prior stage.

6. **No doctor.** There is no way to verify that a project directory is coherent — that the stage matches artifacts on disk, history is intact, and adapters are present.

---

## Decision

Evolve miniV1 with seven targeted additions, delivered as v1.1 Portable Kernel:

| Slice | Addition |
|-------|----------|
| 1 | `minilegion context <tool>` — portable context assembly for Claude, ChatGPT, Copilot, OpenCode |
| 2 | `project-ai/history/` — extract history out of STATE.json into append-only event log |
| 3 | `project-ai/evidence/` — each validate step writes structured evidence bundle |
| 4 | `validate` + `advance` as distinct commands — decouple artifact creation from state advancement |
| 5 | `research --mode brainstorm` — structured exploration with candidate directions + recommendation |
| 6 | `minilegion rollback "<reason>"` — safe stage reset with artifact preservation and history event |
| 7 | `minilegion doctor` — green/yellow/red coherence check across all project files |

All additions are additive and non-breaking. Existing pipeline, approval gates, scope lock, revise loop, atomic writes, and dual JSON/MD artifacts are preserved unchanged.

---

## Consequences

### Positive

- **Portable:** User can resume any project in any AI tool with one command
- **Auditable:** Complete event log in `history/`, validation evidence in `evidence/`
- **Resumable:** `minilegion status` + `minilegion context <tool>` = full context in <2 min
- **Safer:** Validate/advance separation prevents accidental state advancement; rollback enables safe recovery
- **Diagnosable:** Doctor surfaces incoherence before it causes confusing errors

### Negative

- **Richer `project-ai/` structure:** New directories (`history/`, `evidence/`, `context/`, `adapters/`, `templates/`, `memory/`) add surface area to maintain
- **Command surface grows:** 5 new top-level commands (`context`, `history`, `validate`, `advance`, `rollback`, `doctor`)
- **Migration work:** Old STATE.json files with embedded history need non-destructive migration on first access

---

## Rejected Alternatives

**1. Keep as local LLM orchestrator only**
Rejected: The tool is already capable enough to orchestrate multi-tool workflows. The missing piece is portability, not more LLM calls. This option defers the highest-value improvement indefinitely.

**2. Full rewrite**
Rejected: v1.0 has 620 passing tests and a proven architecture. A rewrite would destroy accumulated correctness for marginal gains. Additive evolution is lower risk and faster.

**3. Multi-agent runtime layer**
Rejected: Out of scope for v1.1. Adding a runtime layer would require protocol infrastructure (MCP, tool servers) that is orthogonal to the portability goal. This is a potential v2.0 direction.

---

## Success Criterion

After a 72-hour break from any project, a user runs:

```
minilegion status
minilegion context claude
```

And can resume meaningful work in under 2 minutes, without re-reading the repository or knowing which model was used before.
